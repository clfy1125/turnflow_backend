## 1. 서비스 개요

### 1.1 서비스 정의

TurnFlow는 **AI 기반 링크인바이오(Link-in-Bio) 페이지 빌더** 및 **인스타그램 비즈니스 계정 자동화** SaaS 서비스입니다.

### 1.2 서비스의 본질적 성격

| 구분 | 내용 |
|------|------|
| **서비스 유형** | B2C SaaS (직접 판매형) + 일부 중개 플랫폼 성격 |
| **직접 판매 영역** | 구독제 소프트웨어 이용권, AI 생성 기능, 페이지 빌더 도구 |
| **중개 플랫폼 영역** | 크리에이터가 만든 링크인바이오 페이지를 통해 방문자가 크리에이터의 외부 상품/서비스에 접근 (향후 확장 시) |

> **법적 검토 필요**: 서비스가 순수 SaaS 도구 판매인지, 크리에이터-방문자 중개 플랫폼인지에 따라 책임 범위가 달라지므로 명확한 법적 성격 규정이 필요합니다.

### 1.3 핵심 기능 요약

1. **링크인바이오 페이지 빌더**: 15종의 블록 타입으로 개인/브랜드 페이지 제작 (프로필, 링크, 갤러리, 동영상, 문의폼, 지도, 일정 등)
2. **AI 콘텐츠 생성**: LLM을 활용한 바이오 텍스트 리메이크, 테마 생성, 마케팅 카피 생성
3. **인스타그램 자동화**: 댓글 수집·분류, 키워드 감지, 자동 DM 발송, 성과 대시보드
4. **워크스페이스(팀)**: 다중 사용자가 하나의 워크스페이스에서 협업 가능
5. **페이지 분석**: 방문자 수, 블록별 클릭 수, 유입 경로, 국가별 통계 제공

## 2. 이용자 유형 및 가입 플로우

### 2.1 이용자 구분

| 이용자 유형 | 설명 | 가입 요건 |
|-------------|------|-----------|
| **무료 회원** | 기본 기능 이용 (페이지 3개 제한, AI 생성 불가) | 이메일 기반 회원가입 |
| **유료 회원 (Pro)** | AI 생성, 무제한 페이지, 로고 제거 | 무료 회원 + 구독 결제 |
| **유료 회원 (Pro+)** | Pro 전체 기능 + 커스텀 CSS | 무료 회원 + 구독 결제 |
| **워크스페이스 관리자** | 팀 단위 인스타그램 자동화 기능 이용 | 워크스페이스 생성자 |
| **워크스페이스 멤버** | 초대를 통해 워크스페이스에 참여 | 초대 토큰 수락 |
| **페이지 방문자** | 크리에이터가 만든 페이지를 열람하는 비회원 | 없음 (회원가입 불필요) |

### 2.2 회원가입 및 인증 방식

- **가입 방식**: 이메일 + 비밀번호 기반 (소셜 로그인 미적용)
- **인증 방식**: JWT(JSON Web Token) 기반 인증
  - 액세스 토큰 유효기간: 1시간
  - 리프레시 토큰 유효기간: 7일
- **가입 시 자동 처리**: 무료 요금제(Free Plan) 자동 할당

### 2.3 워크스페이스 역할(RBAC)

| 역할 | 워크스페이스 조회 | 워크스페이스 수정 | 멤버 관리 | 역할 변경 | 워크스페이스 삭제 |
|------|:-:|:-:|:-:|:-:|:-:|
| **Owner (소유자)** | ✅ | ✅ | ✅ | ✅ | ✅ |
| **Admin (관리자)** | ✅ | ✅ | ✅ | ❌ | ❌ |
| **Member (멤버)** | ✅ | ❌ | ❌ | ❌ | ❌ |

## 3. 서비스 기능 상세

### 3.1 링크인바이오 페이지

- **페이지 URL 형식**: `https://도메인/@{사용자슬러그}`
- **블록 타입 (15종)**:

| 블록 분류 | 세부 타입 | 설명 |
|-----------|----------|------|
| 프로필 | profile | 프로필 사진, 헤드라인, 소개글 |
| 단일 링크 | single_link | 외부 URL 버튼 |
| 링크 그룹 | group_link | 다중 링크 폴더 |
| SNS 링크 | social | SNS 아이콘 모음 |
| 동영상 | video | 영상 임베드 |
| 텍스트 | text | 자유 텍스트 |
| 갤러리 | gallery | 이미지 캐러셀 |
| 여백 | spacer | 구분선/간격 |
| 지도 | map | 지도 임베드 |
| 공지 | notice | 알림 배너/팝업 |
| 문의 | inquiry | 콜백 요청 폼 |
| 고객정보 | customer | 고객 정보 수집 폼 |
| 검색 | search | 블록 내 검색 |
| 폴더 | folder | 하위 블록 컨테이너 |
| 일정 | schedule | 이벤트 캘린더 |

- **블록 스케줄링**: 특정 시간대에만 블록 표시/숨김 설정 가능 (`publish_at`, `hide_at`)
- **커스텀 스타일**: 블록별 배경색, 테두리색, 텍스트색, 버튼색 설정 가능
- **커스텀 CSS**: Pro+ 요금제에서만 사용자 직접 CSS 입력 가능

### 3.2 AI 기능

| AI 기능 | 설명 | 소모 토큰 |
|---------|------|-----------|
| 바이오 리메이크 (BIO_REMAKE) | 사용자 컨셉·스타일 기반으로 바이오 텍스트 AI 생성 | 1 토큰/건 |
| 테마 생성 (THEME_GENERATION) | 페이지 테마(색상, 폰트 등) AI 자동 생성 | 1 토큰/건 |
| 카피 생성 (COPY_GENERATION) | 마케팅 카피라이팅 AI 생성 | 1 토큰/건 |

- **사용 LLM 모델**: Gemma-4 (기본), GPT-5.4 (프리미엄)
- **AI 토큰**: 유료 구독 시 월간 토큰 할당, 소진 시 추가 이용 불가
- **AI 작업 상태 흐름**: 대기(queued) → 프롬프트 준비 → 모델 호출 → 응답 파싱 → 이미지 처리 → 완료/실패

### 3.3 인스타그램 자동화 (워크스페이스 기능)

| 기능 | 설명 |
|------|------|
| 계정 연동 | 인스타그램 비즈니스/크리에이터 계정 OAuth 연결 |
| 댓글 수집 | 게시물/릴스의 댓글 자동 수집 |
| AI 댓글 분류 | 관심/스팸/악성/리드 자동 분류 |
| 키워드 감지 | 특정 키워드 포함 댓글 필터링 |
| 자동 DM 발송 | 규칙 기반 자동 다이렉트 메시지 발송 |
| DM 캠페인 | 템플릿 기반 대량 DM 관리 (시간당 최대 200건) |
| 성과 대시보드 | 댓글 수집량, DM 발송량, 성과 지표 시각화 |

### 3.4 페이지 분석(Analytics)

- **수집 데이터**: 페이지 조회수, 블록 클릭수, 유입 경로(Referer), 국가별 통계
- **개인정보 처리**: IP 주소는 SHA-256 해시 처리 후 저장 (원본 IP 미저장)
- **국가 판별**: Cloudflare CF-IPCountry 헤더 기반 (ISO 3166-1 국가 코드)

### 4.3 과금 방식

- **과금 주기**: 월간 정기결제 (매월 자동 갱신)
- **과금 시점**: 최초 결제일 기준 매월 동일일 자동 청구
- **결제 실패 시**: `past_due` 상태로 전환, 자동 재시도 후 지속 실패 시 서비스 제한
- **요금제 변경**: 업그레이드 즉시 적용, 다운그레이드 시 현재 결제 주기 종료 후 적용

### 4.4 요금제 변경에 따른 처리

| 상황 | 처리 방식 |
|------|----------|
| **무료 → 유료 업그레이드** | 즉시 유료 기능 활성화, AI 토큰 할당 |
| **유료 → 무료 다운그레이드** | 결제 주기 종료 후 적용, 3개 초과 페이지 비활성화(`is_active=false`) |
| **유료 간 업그레이드** | 즉시 상위 기능 적용 |
| **구독 해지** | `cancelled` 상태, 현재 결제 주기 종료일까지 서비스 이용 가능 |

## 5. 결제 수단 및 결제 처리

### 5.1 결제 대행사

- **PG사**: PayApp (payapp.kr)
- **결제 방식**: PayApp REST API를 통한 정기결제(Recurring Billing) 처리

### 5.2 지원 결제 수단

| 결제 수단 | 지원 여부 |
|-----------|:---------:|
| 신용카드 | ✅ |
| 휴대전화 소액결제 | x |
| 계좌이체 | x |
| 가상계좌 | x |
| 카카오페이 | x |
| 네이버페이 | x |
| 애플페이 | x |

### 5.3 결제 처리 흐름

```
1. 사용자가 유료 요금제 선택
2. PayApp 정기결제 등록(rebillRegist) API 호출 → 결제 URL 발급
3. 사용자가 결제 URL에서 최초 결제 진행
4. PayApp → TurnFlow 웹훅(feedback)으로 결제 결과 통보
5. 결제 성공 시: 구독 활성화, AI 토큰 할당, 기능 잠금 해제
6. 매월 자동 청구: PayApp rebillCycleMonth API로 정기결제 실행
7. 결제 실패 시: 실패 웹훅 수신 → 재시도 → 지속 실패 시 서비스 제한
```

### 5.4 결제 기록 관리

- **거래 이력(PaymentHistory)**: 금액, 상태(대기/완료/실패/환불), 결제 수단, 결제일시, 영수증 URL 보관
- **웹훅 로그(PayAppWebhookLog)**: 중복 방지를 위한 거래번호+상태 기반 멱등성(Idempotency) 처리
- **정기결제 ID**: `payapp_rebill_no`로 구독과 결제 연결 관리

## 6. 구독 해지 및 환불 정책

### 6.1 구독 해지 프로세스

| 단계 | 설명 |
|------|------|
| 1 | 사용자가 서비스 내 설정에서 구독 해지 요청 |
| 2 | PayApp 정기결제 취소 API 호출 |
| 3 | 구독 상태를 `cancelled`로 변경 |
| 4 | **현재 결제 주기 종료일(`current_period_end`)까지 유료 기능 유지** |
| 5 | 결제 주기 종료 후 무료 플랜으로 자동 전환 |
| 6 | 3개 초과 페이지 비활성화 처리 |

### 6.2 환불 정책

| 조건 | 환불 가능 여부 | 비고 |
|------|:-:|------|
| 유료 전환 후 **7일 이내** | ✅ 전액 환불 | `pro_activated_at` 기준 7일 산정 |
| 유료 전환 후 **7일 초과** | ❌ 환불 불가 | 일반적 SaaS 정책 |
| 구독 해지 | 환불 아님 | 잔여 기간 서비스 이용 후 종료 |
| AI 토큰 사용 후 환불 | 토큰 차감 | 환불 시 부여된 AI 토큰 회수 |

> **법적 검토 필요**: 전자상거래법·콘텐츠산업진흥법상 디지털 콘텐츠 환불 의무 기간, 청약철회 예외 조건(이미 사용한 디지털 콘텐츠), 월 과금 서비스의 일할 계산 환불 의무 여부 검토가 필요합니다.

## 7. 크리에이터 정산 시스템 (향후 도입 예정)

> **참고**: 현재 코드베이스에는 크리에이터 정산 시스템이 구현되어 있지 않습니다. 향후 확장 시 아래 구조가 예상됩니다.

### 7.1 예상 정산 구조

| 항목 | 예상 정책 |
|------|----------|
| **수익 발생원** | 크리에이터 페이지를 통한 상품/서비스 판매 연결 수수료 (향후) |
| **수익 분배 비율** | 미정 (검토 필요) |
| **정산 주기** | 미정 (월 1회 등 검토 필요) |

## 8. 개인정보 및 데이터 처리

### 8.1 수집·보관하는 개인정보

| 정보 유형 | 수집 항목 | 보관 방식 |
|-----------|----------|-----------|
| **회원 정보** | 이메일, 비밀번호(해시), 이름(선택) | PostgreSQL DB 암호화 저장 |
| **인스타그램 토큰** | OAuth 액세스 토큰 | Fernet 대칭키 암호화 (AES 128-bit) |
| **방문자 정보** | IP 해시(SHA-256), 국가 코드, 유입 경로 | 원본 IP 미저장, 해시만 저장 |
| **결제 정보** | 거래 ID, 결제 수단, 금액, 영수증 URL | PayApp PG사 위탁 처리 |
| **AI 작업 이력** | 입력 프롬프트, 생성 결과물 | DB 저장 |

### 8.2 데이터 보호 조치

- **토큰 암호화**: 인스타그램 액세스 토큰은 Fernet(AES 128-bit) 암호화 후 DB 저장
- **비밀번호**: Django 기본 PBKDF2 해싱
- **IP 비식별화**: SHA-256 해시 처리, 원본 IP 미보관
- **접근 제어**: JWT 인증 + 역할 기반 접근 제어(RBAC)
- **통신 보안**: HTTPS 필수 (운영 환경), HSTS 적용
- **데이터 격리**: 워크스페이스 단위 논리적 데이터 분리

### 8.3 외부 전송 데이터

| 전송 대상 | 전송 데이터 | 목적 |
|-----------|-----------|------|
| **Meta(Instagram)** | OAuth 토큰, API 요청 | 인스타그램 기능 연동 |
| **PayApp** | 결제 요청 정보 | 결제 처리 |
| **LLM 서비스** (자체 운영) | 사용자 입력 텍스트 | AI 콘텐츠 생성 |

> **법적 검토 필요**: 개인정보보호법·정보통신망법 기반 개인정보처리방침 별도 작성 필요. 특히 해외 서비스(Meta)로의 개인정보 국외 이전 관련 동의 조항이 필요합니다.
