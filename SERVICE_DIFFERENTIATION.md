## 0. 한눈에 보기 (Executive Summary)

TurnFlow는 **인스타그램 마케팅 퍼널 전체를 하나의 플랫폼**에서 제공한다.

```
게시물/댓글  →  키워드 감지  →  자동 DM(팔로우 게이트·예약)  →  바이오링크 페이지(전환)  →  통계
   └────────────────────── 한 플랫폼, 한 퍼널, 한 청구서 ──────────────────────┘
```

경쟁사는 이 퍼널의 **일부만** 가진다.
- **Linktree / 인포크링크 / 리틀리** → 바이오링크 페이지만. DM 자동화 없음.
- **ManyChat / 국내 DM 자동화 툴** → DM 자동화만. 바이오링크 페이지는 없거나 빈약.

TurnFlow는 둘을 묶고, 각 영역에서도 **AI 페이지 생성**(바이오링크)과 **99.9% 발송 보증 엔진**(DM)으로 차별화한다.

## 1. 핵심 기능 정리

### 1.1 바이오링크 (링크인바이오)

| 영역 | 기능 | 비고 |
|------|------|------|

| **커머스 메타 스크랩** | 쿠팡(파트너스 API)·스마트스토어·오늘의집·네이버쇼핑 링크의 제목/이미지/가격 자동 추출 | 한국 커머스 특화 |
| **공개 페이지** | `/@slug` 공개 URL, 디자인킷(배경·카드·버튼·폰트), 예약 공개 | 본인은 비공개 상태에서도 미리보기 |
| **통계/분석** | 조회수·클릭수·유입 채널·국가 분포, 블록별/링크별 클릭 통계 | `PageView`/`BlockClick` 기반 |
| **편집 이력(스냅샷)** | AI 편집마다 자동 스냅샷, 임의 시점 롤백 | 최대 10건 보존 |
| **이미지 파이프라인** | HEIC→JPEG 등 포맷 정규화, EXIF/GPS 제거, 해상도 상한, 재인코딩 | 개인정보·용량 최적화 |

### 1.2 DM 자동화 / 발송

| 영역 | 기능 | 비고 |
|------|------|------|
| **트리거** | 특정 게시물 / 모든 게시물 / 다음 게시물 자동 부착 / 스토리 답장 | `SPECIFIC_MEDIA`·`ANY_MEDIA`·`NEXT_MEDIA`·`STORY_REPLY` |
| **키워드 매칭** | 포함(ANY) / 전부 포함(ALL) / 완전 일치(EXACT). 비우면 전체 매칭 | 댓글 텍스트 기준 |
| **발송 엔진** | `QUEUED → SUBMITTING → ACCEPTED → DELIVERED → READ` 상태머신 | 각 단계 신호 추적 = 관측성 |
| **도착 이중 확인** | echo 웹훅 매칭 + Conversations API 능동 조회 | "보냈는데 실제로 안 감"을 잡아냄 |
| **멱등성** | `idempotency_key`(sha256) + `EventInbox` 장부 | 웹훅 재전송·재시도에도 중복 발송 0 |
| **실패 명시 분류** | `FAILED_TOKEN`/`FAILED_WINDOW`/`FAILED_PARAM`/`RATE_LIMITED`/`FAILED_NO_TRACE` | Meta 에러코드를 사용자 액션으로 번역 |
| **24h 정책 검증** | Private Reply 24시간 윈도우 만료 시 `FAILED_WINDOW`로 분류·재시도 중단 | Meta 정책 준수 내장 |
| **팔로우 게이트** | "팔로우했어요" 버튼 → 팔로우 여부 silent 검증 → 보상 DM 발송 | 팔로워 전환 자동화 |
| **공개 답글** | 댓글에 공개 답글을 배치+쿨다운으로 게시(봇 감지 회피) | 템플릿 무작위 선택 |
| **안전속도 거버너** | IG 계정 단위 분당/시간당 발송 상한(플랜별) | 계정 정지 리스크 방어 |
| **Mock 모드** | `INSTAGRAM_MOCK_MODE`로 운영 승인 전에도 파이프라인 테스트 | 개발/QA 가속 |

### 1.3 DM 예약 / 캠페인 발송 (구현 완료)

> 캠페인에 **시작일·종료일**을 지정하면, 지정 기간에만 자동으로 활성화되고 종료 시각에 자동 종료된다. 별도 수동 on/off 불필요.

| 필드 | 설명 |
|------|------|
| `scheduled_start_at` | 발송 시작 시각. 비우면 즉시(수동 운영). |
| `scheduled_end_at` | 자동 종료 시각. 비우면 무기한. |
| 예약 윈도우 필터 | `시작 없음/지남` AND `종료 없음/안 지남` AND `status=ACTIVE`인 캠페인만 실제 발송 경로에 노출 |
| 자동 종료 | Celery Beat `enforce_campaign_schedules`가 **1분 주기**로 종료 시각 지난 ACTIVE 캠페인을 `COMPLETED`로 전환 (멱등) |

상세 스펙·API·동작은 [5. DM 예약/캠페인 발송 기능 명세](#5-dm-예약캠페인-발송-기능-명세) 참고.

> - 무료 플랜만으로도 페이지·블록 15종·커스텀 CSS·통계·**DM 자동화 500건/월**을 제공 → 진입 장벽이 사실상 0.

## 3. DM 자동화 경쟁사 비교

### 3.1 가격 모델 비교 (2026-06 기준)

| 서비스 | 무료 | 유료 | 과금 모델 | 위험 |
|--------|------|------|----------|------|
| **TurnFlow** | **DM 500건/월** 무료 | **프로 월 ₩14,900 — DM 자동화·AI·스팸필터 무제한** | **월 정액 (무제한)** | 없음 — 성장해도 ₩14,900 고정 |
| ManyChat | 25 컨택트·2채널·4 자동화 | $14(Essential) / **$29(Pro, 약 ₩4만)** / $69(Business) | **활성 컨택트 종량제** | 청중↑ → 비용↑ (성공할수록 비싸짐) |
| 국내 DM 자동화 툴 | 천차만별 | — | 발송량/계정 단위 | 발송 신뢰성·관측성 편차 큼 |

> **핵심**: ManyChat은 *활성 컨택트 수*에 비례해 과금한다. 캠페인이 잘될수록 청구액이 계단식으로 오르며 "월 $25로 시작 → 8개월 뒤 $345"가 흔한 패턴(출처 참고). 반면 **TurnFlow 프로는 월 ₩14,900 고정에 DM 자동화 무제한** — 발송량이 폭증해도 비용이 그대로다. 게다가 무료 플랜만으로도 **DM 500건/월**을 주는데, 이는 ManyChat 무료(컨택트 25개)와 비교 불가한 폭이다.

### 3.2 기능/성능 비교

| 항목 | TurnFlow | ManyChat | 국내 툴(일반) |
|------|:--------:|:--------:|:-----------:|
| 댓글→DM 자동화 | ✅ | ✅ | ✅ |
| 키워드 매칭(ANY/ALL/EXACT) | ✅ | ✅ | 부분 |
| 스토리 답장 트리거 | ✅ | ✅ | 부분 |
| **발송 보증 상태머신** | ✅ (QUEUED→READ) | 제한적 | ❌ |
| **도착 이중 확인(echo+API)** | ✅ | ❌ | ❌ |
| **멱등성(중복발송 방지)** | ✅ | 부분 | ❌ |
| **실패 명시 분류** | ✅ (5종) | 부분 | ❌ |
| 24h 정책 검증 내장 | ✅ | ✅ | 부분 |
| 팔로우 게이트 | ✅ | ✅ (상위플랜) | 부분 |
| 계정 안전속도 거버너 | ✅ | 부분 | ❌ |
| **예약/캠페인(시작·종료일)** | ✅ | ✅ | 부분 |
| **바이오링크 통합** | ✅ | ❌ | ❌ |
| 한국어/원화 결제 | ✅ | ❌(영문·USD) | ✅ |

**DM 요약**: 기능 *개수*만 보면 ManyChat이 글로벌 1위지만, TurnFlow의 차별점은 **발송 신뢰성과 관측성**이다. 상태머신·멱등성·이중 도착 확인·실패 분류로 "보낸 줄 알았는데 누락"을 구조적으로 제거한다. 국내 툴 대비로는 이 엔진 자체가 격차이고, ManyChat 대비로는 가격 예측성과 바이오링크 통합이 격차다.

> **솔직한 한계**: 현재 댓글 의도 분석은 **키워드 기반**이며 LLM 기반 의도 분석은 미구현이다(로드맵). ManyChat 대비 플로우 빌더의 시각적 분기 자유도도 아직 단순하다. 단, 바이오링크 쪽 AI(페이지 생성)는 경쟁사를 크게 앞선다.

## 4. TOP 5 차별점 (상세)

### 1️⃣ 올인원 퍼널 — 바이오링크 + DM 자동화 통합
경쟁사는 퍼널의 한 조각만 판다. Linktree·인포크·리틀리는 "페이지", ManyChat·국내 툴은 "DM"이다.
TurnFlow는 **댓글 감지 → 자동 DM(팔로우 게이트) → 바이오링크 페이지 전환 → 통계**를 한 플랫폼·한 청구서로 묶는다.
→ 고객은 두 개 툴을 구독·연동·관리할 필요가 없고, 데이터가 한 곳에 모여 전환 분석이 가능하다.

### 2️⃣ AI 페이지 생성·리뉴얼 + 경쟁사 임포트
컨셉 한 줄이면 12개 업종 레시피로 완성형 페이지가 나오고, **디자인 가드**(대비·명도·슬롭 색 자동 보정)와 **비전 비평 루프**가 "디자인이 되는" 결과를 보증한다. 경쟁사엔 없는 수준이다.
결정적으로 **Linktree·인포크·리틀리 URL을 붙여넣으면 그대로 이주**된다 → 경쟁사 락인을 무력화하는 전환 무기.

### 3️⃣ 99.9% DM 발송 보증 엔진
`QUEUED → SUBMITTING → ACCEPTED → DELIVERED → READ` 상태머신, `idempotency_key`+`EventInbox` 멱등성, **echo 웹훅 + Conversations API 이중 도착 확인**, 그리고 Meta 에러를 `FAILED_TOKEN/WINDOW/PARAM/RATE_LIMITED/NO_TRACE`로 번역하는 실패 분류.
→ "보낸 줄 알았는데 누락"이라는 자동화 툴의 고질병을 *구조적으로* 제거한다. 국내 툴과의 격차이자, ManyChat과도 차별되는 관측성.

### 4️⃣ 무제한 정액제 — 프로 월 ₩14,900
- **vs ManyChat**: 컨택트 종량제(성공할수록 비싸짐, $29~$69+) 대신 **월 ₩14,900 고정에 DM 자동화·AI 무제한**. 발송이 폭증해도 비용이 그대로다. 무료 플랜도 DM 500건/월(ManyChat 무료=컨택트 25개).
- **vs Linktree**: ₩14,900은 Linktree Pro($15≈₩2만)보다 싸면서 페이지 5개 + AI 무제한 + DM 자동화까지 포함. 커머스 거래 수수료 **0%**(Linktree는 무료 12%·유료 9%) → 공구·판매 셀러에게 직접 이득.
- **원화·PayApp 정기결제**로 부가세·세금계산서 등 국내 결제 환경에 맞다.

### 5️⃣ 한국 시장 100% 현지화
카카오톡 채널·네이버 예약·스마트스토어·쿠팡 파트너스 네이티브 연동, 한국어 AI 카피, **청첩장·공동구매·재능 커미션** 등 국내 특화 카테고리, 화이트리스트 한글 폰트.
운영 측면에서도 24h 메시징 정책 검증과 **계정 단위 안전속도 거버너**로 인스타 계정 정지 리스크를 낮춘다 — 글로벌 툴이 신경 쓰지 않는 국내 사용자 보호.

## 5. DM 예약/캠페인 발송 기능 명세

> 캠페인을 "언제부터 언제까지" 자동으로 돌릴지 지정하는 기능. 시작 전엔 발송 안 됨, 종료 시각엔 자동으로 멈춤.

### 5.1 데이터 모델 (`AutoDMCampaign`)

| 필드 | 타입 | 의미 | 기본 |
|------|------|------|------|
| `scheduled_start_at` | datetime(nullable) | 캠페인 발송 시작 시각. `null`이면 즉시(상태값 수동 운영). | `null` |
| `scheduled_end_at` | datetime(nullable) | 자동 종료 시각. `null`이면 무기한. | `null` |
| `status` | enum | `ACTIVE`/`PAUSED`/`COMPLETED` 등 | — |

**발송 노출 조건(예약 윈도우 필터, `schedule_window_q()`)**
```
(scheduled_start_at IS NULL OR scheduled_start_at <= now)
AND (scheduled_end_at IS NULL OR scheduled_end_at >  now)
AND status = ACTIVE
```
→ 위 조건을 만족하는 캠페인만 댓글/웹훅 처리 경로에서 실제 DM을 발송한다.

### 5.2 자동 종료 (Celery Beat)

- 태스크: `enforce_campaign_schedules` (`apps/integrations/tasks.py`)
- 주기: **1분** (`CELERY_BEAT_SCHEDULE` 등록)
- 동작: `scheduled_end_at <= now` 인 `ACTIVE` 캠페인을 `COMPLETED`로 일괄 전환. 멱등(여러 번 돌아도 안전).

> ⚠️ 운영 메모: Beat 전용 컨테이너가 compose에 없으므로 예약 자동 종료를 쓰려면 Beat 서비스가 떠 있어야 함(또는 `django-celery-beat`). 미가동 시 종료 시각이 지나도 자동 전환되지 않음 — 배포 체크리스트에 포함할 것.

### 5.3 API 사용 예시

**캠페인 생성 (예약 포함)**
```bash
curl -X POST https://dev-api.turnflow.link/api/v1/integrations/auto-dm-campaigns/ \
  -H "Authorization: Bearer <ACCESS_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "여름 공구 캠페인",
    "trigger_type": "ANY_MEDIA",
    "keyword_filter": ["구매", "링크"],
    "keyword_mode": "ANY",
    "message_template": "안녕하세요! 신청 링크 보내드려요 👉 ...",
    "status": "ACTIVE",
    "scheduled_start_at": "2026-07-01T09:00:00+09:00",
    "scheduled_end_at":   "2026-07-14T23:59:59+09:00"
  }'
```

**응답 (201)**
```json
{
  "success": true,
  "data": {
    "id": "…",
    "name": "여름 공구 캠페인",
    "status": "ACTIVE",
    "scheduled_start_at": "2026-07-01T00:00:00Z",
    "scheduled_end_at": "2026-07-14T14:59:59Z",
    "is_within_schedule": false
  }
}
```

**검증 규칙(시리얼라이저)**
- `scheduled_end_at`은 `scheduled_start_at`보다 미래여야 함 → 위반 시 400 + 통일 에러 포맷
  ```json
  { "success": false, "error": { "code": 400, "message": "종료 시각은 시작 시각보다 이후여야 합니다.", "details": {} } }
  ```

### 5.4 사용자 시나리오

1. 셀러가 7/1~7/14 공구 캠페인을 미리 등록(시작/종료 지정) → 출시 전엔 절대 발송되지 않음.
2. 7/1 09:00부터 댓글 키워드 매칭 시 자동 DM 시작.
3. 7/14 23:59 종료 시각이 지나면 Beat가 캠페인을 `COMPLETED`로 전환 → 이후 댓글에는 발송 안 됨.
4. 별도 수동 on/off나 야간 대기 불필요. "걸어두고 잊는" 운영.

## 7. 정직성 메모 (내부용)

문서의 신뢰도를 위해 과장하지 않은 부분을 명시한다.

- ✅ **실제 구현됨**: 바이오링크 블록/AI 생성·리뉴얼/디자인 가드/비전 비평/외부 임포트/커머스 메타/통계/스냅샷, DM 상태머신·멱등성·이중 확인·실패 분류·24h 검증·팔로우 게이트·공개답글·안전속도, **DM 예약/캠페인(시작·종료일)**.
- ⚠️ **미구현/로드맵**: DM 측 **LLM 기반 댓글 의도 분석**(현재 키워드 기반), 시각적 플로우 빌더의 복잡 분기, 댓글 자동 분류(`CommentClassification`).
- 📝 **확인 필요**: 경쟁사 가격은 2026-06 공개값으로 수시 변동. "99.9%"는 설계 목표/내부 지표 명칭이며 대외 표기 시 측정 근거를 함께 제시할 것.
- 🔧 **코드 동기화 필요**: 본 문서의 요금제는 출시 요금제(무료/프로)를 기준으로 한다. 그러나 코드의 `apps/billing/models.py` `PlanLimits`는 아직 레거시 3단(`Starter/Pro/Enterprise`, DM 100/1,000/무제한)으로 정의돼 있다. 출시 플랜(무료=DM 500/월, 프로=무제한)과 불일치하므로 `PlanLimits`·`PlanChoices`·`rate_governor` 키를 출시 요금제에 맞춰 정리할 것.
