## 3. 핵심 발견

### (A) 댓글 정렬이 복원율의 최대 레버 — 22배
IG `/comments` 는 **최신순 고정**이며 `order=chronological` 이 무시된다(실측: 4월 게시물의
첫 페이지 댓글이 7월). 캠페인은 게시 직후 돌아가 **"초기(오래된) 댓글러"가 DM을 받는다.**
따라서 첫 페이지(최신 댓글러)만 조회하면 대부분 DM 미수신자다.
→ 끝까지 페이징한 tail 의 초기 댓글러 기준 복원율 **38% vs 최신 1.7%(약 22배)**.
수정: `collect.fetch_oldest_commenters` 로 tail 을 표본, `_stage_targeted_dms` 가 이를 사용.

### (B) 타겟 DM 조회로 게시물↔발신DM 정확 링크
**댓글 `from.id` == 메시징 `user_id` 로 동일 ID 공간임을 실측 증명**(원 계획서의
"id-space 불일치" 우려 기각). 댓글러 IGSID 로 `GET /{ig}/conversations?user_id=` 직접 조회 →
3만+ 대화방 전체 페이징 없이 게시물별 발신 DM 을 정확히 링크.

### (C) 복원 불가의 구조적 바닥 = 바이럴 게시물의 비팔로워 미전달
감지된 캠페인 중 일부(댓글 600~4935개의 **바이럴 게시물**)는 복원에 실패했다.
"우리 페이징 캡이 초기 댓글러를 못 봐서"라는 가설을 검증하려 상위 3개를 **런칭일까지
깊이 파봤으나(최대 80페이지·4000댓글, 게시일 근처 도달) 초기 댓글러조차 DM 적중 0~1/15**.

→ 원인은 페이징이 아니라, **바이럴 게시물은 도달이 커서 댓글러 대부분이 비팔로워** →
자동 DM(Private Reply)이 **채널 미개설로 애초에 전달 실패**(대화 자체가 없음) → 정렬·깊이와
무관하게 복원 불가. 반대로 **작고 타게팅된 게시물**은 팔로워 댓글러 비중이 높아 잘 복원된다.
**깊은 페이징으로도 못 살리므로 캡을 올리는 건 API 낭비**(현재 캡이 합리적).
